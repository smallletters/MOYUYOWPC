## 二、开发环境搭建

### 2.1 克隆仓库

```bash
git clone https://github.com/smallletters/moyuyo-app.git
cd moyuyo-app
```

### 2.2 安装依赖

```bash
# 推荐使用 pnpm
pnpm install

# 或
npm install
```

### 2.3 环境要求

| 工具           | 版本要求        | 验证命令                 |
| -------------- | --------------- | ------------------------ |
| Node.js        | >= 24.16.0      | `node -v`                |
| npm / pnpm     | 最新            | `pnpm -v`                |
| HBuilderX      | 最新版（可选）  | App 端真机运行需要       |
| Android Studio | 最新版          | Android 原生插件编译需要 |
| Xcode          | 最新版（macOS） | iOS 原生插件编译需要     |

### 2.4 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 填入实际后端 API 地址：

```env
# 后端 API 基础地址（开发环境指向本地 moyuyo-server）
VITE_API_BASE=http://localhost:8080/api

# 生产环境使用部署地址
# VITE_API_BASE=https://api.moyuyo.com
```

### 2.5 可选工具

| 工具                 | 用途                  |
| -------------------- | --------------------- |
| **Postman / Apifox** | 接口调试              |
| **Vue DevTools**     | 调试 Pinia 状态和组件 |
| **微信开发者工具**   | 微信小程序端预览      |

## 六、开发流程

### 6.1 Git 分支策略

- **main** — 生产分支，只允许从 develop 合入
- **develop** — 开发主分支
- **feature/xxx** — 功能分支，从 develop 拉出，合回 develop
- **fix/xxx** — Bug 修复分支

### 6.2 提交流程

1. `git checkout -b feature/xxx develop`
2. 编码并本地测试
3. `git add <files>` + `git commit`（遵循 Conventional Commits）
4. `git push origin feature/xxx`
5. GitHub 创建 Pull Request → develop
6. 至少 1 人 Review 通过后合入

### 6.3 提交信息格式

```
<type>(<scope>): <description>

类型: feat / fix / refactor / style / docs / chore / test
示例: feat(cart): 添加购物车商品数量编辑功能
```

## 七、原生插件说明

项目通过 UTSDK 桥接 8 个原生能力模块：

| 插件           | 能力                      | 调用方式                                            |
| -------------- | ------------------------- | --------------------------------------------------- |
| **支付**       | Apple Pay / Google Pay    | `import { createPay } from '@/plugins'`             |
| **社交登录**   | Apple / Google / Facebook | `import { loginWithApple } from '@/plugins'`        |
| **Pet Hub 3D** | 3D 模型加载 / 场景切换    | `import { openPetHub3d } from '@/plugins'`          |
| **AR 试穿**    | ARKit / ARCore            | `import { startArTryOn } from '@/plugins'`          |
| **推送**       | APNs / FCM                | `import { registerPush } from '@/plugins'`          |
| **相机**       | 拍照 / 相册选择 / 压缩    | `import { takePhoto } from '@/plugins'`             |
| **定位**       | 定位 / 地图选择           | `import { getCurrentLocation } from '@/plugins'`    |
| **生物识别**   | Face ID / 指纹            | `import { authenticateBiometric } from '@/plugins'` |

协议定义见 [`src/plugins/types.ts`](file:///d:/MOYUYOWPC/moyuyo-app/src/plugins/types.ts)。

> **H5 环境**：原生插件自动降级为 Web 实线或提示不支持，不影响开发调试。

## 十、常见问题

**Q: H5 端运行正常，但原生插件报错？**  
A: 原生插件仅在 APP 端可用，H5 端已做降级处理，不影响功能开发。

**Q: 后端接口返回 401？**  
A: 检查 Token 是否过期，登录后会自动刷新。也可在 `.env` 中关闭鉴权（开发阶段临时）。

**Q: 小程序端白屏或接口报错？**  
A: 小程序 require 仅支持 HTTPS 和 `localhost`，确保开发时使用 `localhost` 地址。
