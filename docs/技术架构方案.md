## 一、架构总览

### 1.1 设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                        MOYUYO 架构设计原则                       │
├─────────────────────────────────────────────────────────────────┤

```

### 2.2 MVP 阶段架构策略

> **原则：MVP 阶段优先单体部署，随业务增长逐步拆分微服务。**

| 阶段 | 架构形态 | 服务数量 | 触发条件 |
| :--- | :--- | :---: | :--- |
| **MVP（V1.0）** | 模块化单体（Modular Monolith） | 1 个应用 | 日活 < 10 万 |
| **增长期（V1.1）** | 核心服务拆分（用户/商品/订单/支付独立） | 4~6 个 | 日活 10~100 万 |
| **成熟期（V2.0）** | 完全微服务化 | 10+ 个 | 日活 > 100 万，多团队协作 |

### 2.3 缓存架构（Redis）

#### 2.3.1 缓存分层策略

| 层级 | 缓存位置 | 缓存内容 | TTL | 更新策略 |
| :--- | :--- | :--- | :--- | :--- |
| **L1 本地缓存** | Caffeine（JVM 内） | 配置字典、枚举值、不变数据 | 10~30 分钟 | 配置变更主动失效 |
| **L2 分布式缓存** | Redis Cluster | 商品详情、用户信息、库存、活动配置 | 5~60 分钟 | 写操作更新 + 主动失效 |
| **L3 数据库** | MySQL | 全量持久化数据 | 永久 | 最终一致 |

#### 2.3.2 关键缓存设计

```
缓存 Key 命名规范：{业务域}:{对象标识}:{属性}

示例：
- product:sku:{skuId}:info          → SKU 基础信息（30min）
- product:sku:{skuId}:stock         → SKU 实时库存（5min，Lua 原子扣减）
- user:{userId}:profile             → 用户档案（30min）
- marketing:seckill:{activityId}    → 秒杀活动配置（5min）
- marketing:coupon:{couponId}:stock → 优惠券库存（1min，分布式锁）
- order:cart:{userId}               → 购物车（永久，跟随用户）
```

#### 2.3.3 缓存三防策略

| 问题 | 解决方案 |
| :--- | :--- |
| **缓存击穿** | 热点 Key 加互斥锁（分布式锁），永不过期 + 后台异步更新 |
| **缓存雪崩** | TTL 加随机偏移（±10%），多级缓存，限流降级 |
| **缓存穿透** | 布隆过滤器拦截不存在的 Key，空值缓存（短 TTL 30s） |

### 2.4 消息队列架构（RocketMQ）

#### 2.4.1 Topic 规划

| Topic | 生产者 | 消费者 | 消息类型 | 用途 |
| :--- | :--- | :--- | :--- | :--- |
| `order_created` | 订单服务 | 营销/会员/物流/通知 | 普通消息 | 订单创建后触发后续流程 |
| `order_paid` | 支付服务 | 订单/物流/会员/通知 | 事务消息 | 支付成功后更新订单状态 |
| `order_cancelled` | 订单服务 | 库存/营销/通知 | 普通消息 | 订单取消后释放库存 |
| `payment_refund` | 支付服务 | 订单/财务/通知 | 普通消息 | 退款处理通知 |
| `user_registered` | 用户服务 | 营销/会员/通知 | 普通消息 | 新用户注册发欢迎礼 |
| `points_changed` | 会员服务 | 通知/数据统计 | 普通消息 | 积分变动通知 |
| `seckill_order` | 秒杀网关 | 订单服务 | 延迟消息 | 秒杀订单超时检查 |
| `logistics_sync` | 物流服务 | 订单/通知 | 普通消息 | 物流状态变更同步 |
| `data_export` | GDPR 模块 | 数据导出服务 | 异步消息 | 用户数据导出任务 |

#### 2.4.2 消息可靠性保障

- **生产端**：同步发送 + 重试机制（最多 3 次），失败落本地消息表
- **消费端**：幂等消费（去重表 / Redis 幂等键），失败重试 + 死信队列
- **事务消息**：支付、订单等关键流程使用 RocketMQ 事务消息保证最终一致

### 2.5 熔断降级方案（Sentinel）

#### 2.5.1 限流规则

| 资源名 | 限流模式 | 阈值类型 | 阈值 | 降级策略 |
| :--- | :--- | :--- | :--- | :--- |
| `/api/auth/login` | QPS | 单机 | 600/min | 快速失败 |
| `/api/order/create` | QPS | 单机 | 300/min | 快速失败 |
| `/api/payment/*` | QPS | 集群 | 500/min | 快速失败 |
| `/api/seckill/*` | QPS | 集群 | 2000/min | 排队等待（最多 1000 队列） |
| `inventory_service` | 线程数 | 单机 | 100 | 半降级（降级 50%） |
| `payment_service` | 线程数 | 单机 | 50 | 快速失败 |

#### 2.5.2 熔断规则

| 资源名 | 熔断策略 | 最大并发 | 降级条件 | 降级策略 |
| :--- | :--- | :--- | :--- | :--- |
| `mysql_read` | 错误比率 | 100 | 错误率 > 50% 持续 10s | 返回缓存数据 |
| `mysql_write` | 错误次数 | 50 | 连续错误 > 20 次 | 快速失败 |
| `redis_service` | 响应时间 | 200 | P95 > 200ms 持续 10s | 返回默认值 |
| `payment_gateway` | 错误比率 | 50 | 错误率 > 30% 持续 30s | 返回"稍后重试" |
| `mq_producer` | 错误比率 | 100 | 错误率 > 80% 持续 5s | 写入本地消息表 |

#### 2.5.3 降级策略

| 降级类型 | 触发条件 | 处理方式 |
| :--- | :--- | :--- |
| **快速失败** | 超过限流阈值 | 立即返回 `429 Too Many Requests` |
| **排队等待** | 超过限流阈值但未达队列上限 | 排队等待，最多等待 5s |
| **半降级** | 部分请求降级 | 50% 请求正常处理，50% 返回降级数据 |
| **缓存兜底** | 依赖服务不可用 | 返回缓存中的数据 |
| **默认值兜底** | 依赖服务不可用 | 返回预设的默认值 |
| **服务降级** | 非核心服务不可用 | 跳过该服务，继续主流程 |

#### 2.5.5 熔断降级流程

```
请求进入
    ↓
Sentinel 限流检查
    ↓
[通过] → 正常处理
    ↓
[拒绝] → 降级策略处理
    ↓
返回降级响应

熔断状态流转：
CLOSED → OPEN（错误率超标）→ HALF-OPEN（尝试恢复）→ CLOSED（恢复成功）
```

#### 2.6.2 支付通道回调处理差异

| 通道 | 回调类型 | 签名方式 | 重试机制 | 状态通知 |
| :--- | :--- | :--- | :--- | :--- |
| **Stripe** | Webhook | Stripe-Signature Header | 自动重试（3次） | `payment_intent.succeeded` |
| **PayPal** | IPN + Webhook | 事件签名 + 证书验证 | IPN 自动重试，Webhook 需自行实现 | `PAYMENT.SALE.COMPLETED` |
| **Apple Pay** | Stripe Webhook | Stripe-Signature | 自动重试（3次） | `payment_intent.succeeded` |
| **Google Pay** | Stripe Webhook | Stripe-Signature | 自动重试（3次） | `payment_intent.succeeded` |
| **Klarna** | Klarna Webhook | HMAC 签名 | 自动重试（最多 5 次） | `order_completed` |
| **iDEAL** | Stripe Webhook | Stripe-Signature | 自动重试（3次） | `source.chargeable` |
| **Affirm** | Affirm Webhook | API Key 签名 | 需自行实现 | `checkout.completed` |

**回调处理流程**：

```
回调请求
    ↓
签名验证
    ↓
[验证失败] → 记录日志 + 返回 400
    ↓
[验证成功] → 获取订单号 + 幂等检查
    ↓
[已处理] → 返回 200（幂等）
    ↓
[未处理] → 更新订单状态 + 发送 MQ 消息 + 返回 200
```

**统一回调处理接口**：

```java
// PaymentCallbackHandler.java
public interface PaymentCallbackHandler {
    // 验证签名
    boolean verifySignature(HttpServletRequest request);
    
    // 解析回调数据
    PaymentCallback parseCallback(HttpServletRequest request);
    
    // 处理回调
    void handle(PaymentCallback callback);
    
    // 获取通道类型
    PaymentChannel getChannel();
}
```

### 3.4 网络请求封装

**核心设计要点**：
- 统一请求拦截器（Header 注入、Token 附加、签名生成）
- 统一响应拦截器（错误码处理、Token 过期自动刷新、业务错误提示）
- 支持请求取消、超时控制、重试机制
- 支持文件上传/下载进度监听
- 请求重试策略（指数退避）
- 请求并发控制

**完整请求封装设计**：

```typescript
// src/api/request.ts
interface RequestConfig {
  url: string
  method: 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH'
  data?: Record<string, any>
  params?: Record<string, any>
  headers?: Record<string, string>
  timeout?: number
  retryCount?: number
  showLoading?: boolean
  showError?: boolean
}

interface ResponseData<T = any> {
  code: number
  message: string
  data: T
  timestamp: number
}

class HttpRequest {
  private baseURL = process.env.VUE_APP_API_BASE_URL || '/api'
  private timeout = 30000
  private retryCount = 3
  private retryDelay = 1000
  
  // 请求拦截器
  private requestInterceptor(config: RequestConfig): RequestConfig {
    const token = storage.get<string>('access_token')
    
    return {
      ...config,
      url: `${this.baseURL}${config.url}`,
      timeout: config.timeout || this.timeout,
      headers: {
        'Content-Type': 'application/json',
        'X-Requested-With': 'XMLHttpRequest',
        'X-Timestamp': Date.now().toString(),
        'X-Signature': this.generateSignature(config),
        ...(token && { 'Authorization': `Bearer ${token}` }),
        ...config.headers
      }
    }
  }
  
  // 响应拦截器
  private async responseInterceptor(response: any): Promise<any> {
    const { statusCode, data } = response
    
    if (statusCode === 200) {
      if (data.code === 401) {
        await this.refreshToken()
        return this.request(response.config)
      }
      
      if (data.code !== 0) {
        if (response.config.showError !== false) {
          uni.showToast({ title: data.message, icon: 'error' })
        }
        throw new Error(data.message)
      }
      
      return data.data
    }
    
    throw new Error(`HTTP ${statusCode}`)
  }
  
  // Token 刷新
  private async refreshToken(): Promise<void> {
    const refreshToken = storage.get<string>('refresh_token')
    if (!refreshToken) {
      uni.navigateTo({ url: '/pages/login/index' })
      throw new Error('请重新登录')
    }
    
    try {
      const response = await uni.request({
        url: `${this.baseURL}/api/auth/refresh`,
        method: 'POST',
        data: { refresh_token: refreshToken }
      })
      
      const { access_token, refresh_token: newRefreshToken } = response.data.data
      storage.set('access_token', access_token)
      storage.set('refresh_token', newRefreshToken)
    } catch {
      storage.remove('access_token')
      storage.remove('refresh_token')
      uni.navigateTo({ url: '/pages/login/index' })
      throw new Error('登录已过期，请重新登录')
    }
  }
  
  // 生成签名
  private generateSignature(config: RequestConfig): string {
    const timestamp = config.headers?.['X-Timestamp'] || Date.now().toString()
    const secret = process.env.VUE_APP_API_SECRET || ''
    const str = `${config.method}&${config.url}&${timestamp}&${secret}`
    return crypto.sha256(str)
  }
  
  // 指数退避重试
  private async requestWithRetry(config: RequestConfig): Promise<any> {
    let lastError: Error | null = null
    
    for (let i = 0; i <= (config.retryCount || this.retryCount); i++) {
      try {
        return await this.request(config)
      } catch (error) {
        lastError = error as Error
        
        if (i < (config.retryCount || this.retryCount)) {
          await new Promise(resolve => setTimeout(resolve, this.retryDelay * Math.pow(2, i)))
        }
      }
    }
    
    throw lastError
  }
  
  // 核心请求方法
  async request<T>(config: RequestConfig): Promise<T> {
    const finalConfig = this.requestInterceptor(config)
    
    if (finalConfig.showLoading !== false) {
      uni.showLoading({ title: 'Loading...' })
    }
    
    try {
      const response = await uni.request(finalConfig)
      return this.responseInterceptor(response)
    } finally {
      if (finalConfig.showLoading !== false) {
        uni.hideLoading()
      }
    }
  }
  
  // GET 请求
  get<T>(url: string, params?: Record<string, any>, config?: Omit<RequestConfig, 'url' | 'method'>): Promise<T> {
    return this.requestWithRetry({ ...config, url, method: 'GET', params })
  }
  
  // POST 请求
  post<T>(url: string, data?: Record<string, any>, config?: Omit<RequestConfig, 'url' | 'method'>): Promise<T> {
    return this.requestWithRetry({ ...config, url, method: 'POST', data })
  }
  
  // PUT 请求
  put<T>(url: string, data?: Record<string, any>, config?: Omit<RequestConfig, 'url' | 'method'>): Promise<T> {
    return this.requestWithRetry({ ...config, url, method: 'PUT', data })
  }
  
  // DELETE 请求
  delete<T>(url: string, params?: Record<string, any>, config?: Omit<RequestConfig, 'url' | 'method'>): Promise<T> {
    return this.requestWithRetry({ ...config, url, method: 'DELETE', params })
  }
  
  // 文件上传
  upload<T>(url: string, filePath: string, formData?: Record<string, any>): Promise<T> {
    return new Promise((resolve, reject) => {
      uni.uploadFile({
        url: `${this.baseURL}${url}`,
        filePath,
        name: 'file',
        formData,
        header: {
          'Authorization': `Bearer ${storage.get<string>('access_token')}`
        },
        success: (response) => {
          try {
            const data = JSON.parse(response.data)
            if (data.code === 0) {
              resolve(data.data)
            } else {
              reject(new Error(data.message))
            }
          } catch {
            reject(new Error('Upload failed'))
          }
        },
        fail: (error) => reject(error)
      })
    })
  }
}

export const http = new HttpRequest()
```

| `useCartStore` | 购物车商品、数量、选中状态 | ✅ | items, selectedIds |
| `useAppStore` | 全局配置、主题、语言、网络状态 | ✅ | theme, language, networkStatus, version |

**useUserStore 设计**：

```typescript
// src/store/user.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { storage } from '@/utils/storage'
import { userApi } from '@/api/user'

export const useUserStore = defineStore('user', () => {
  const userInfo = ref<any>(null)
  const accessToken = ref<string>('')
  const refreshToken = ref<string>('')
  
  const isLoggedIn = computed(() => !!accessToken.value)
  
  function login(token: string, refresh: string, user: any) {
    accessToken.value = token
    refreshToken.value = refresh
    userInfo.value = user
    
    storage.set('access_token', token, { encrypt: true })
    storage.set('refresh_token', refresh, { encrypt: true })
    storage.set('user_info', user, { encrypt: true })
  }
  
  function logout() {
    userInfo.value = null
    accessToken.value = ''
    refreshToken.value = ''
    
    storage.remove('access_token')
    storage.remove('refresh_token')
    storage.remove('user_info')
    
    uni.navigateTo({ url: '/pages/login/index' })
  }
  
  async function fetchProfile() {
    const data = await userApi.getProfile()
    userInfo.value = data
    storage.set('user_info', data, { encrypt: true })
    return data
  }
  
  function loadFromStorage() {
    accessToken.value = storage.get<string>('access_token') || ''
    refreshToken.value = storage.get<string>('refresh_token') || ''
    userInfo.value = storage.get<any>('user_info') || null
  }
  
  return {
    userInfo,
    accessToken,
    refreshToken,
    isLoggedIn,
    login,
    logout,
    fetchProfile,
    loadFromStorage
  }
})

  })
  
  const totalCount = computed(() => {
    return items.value.reduce((sum, item) => sum + item.quantity, 0)
  })
  
  const selectedCount = computed(() => {
    return items.value.filter(item => item.selected).reduce((sum, item) => sum + item.quantity, 0)
  })
  
  function addItem(item: Omit<CartItem, 'selected'>) {
    const existing = items.value.find(i => i.skuId === item.skuId)
    
    if (existing) {
      existing.quantity += item.quantity
    } else {
      items.value.push({ ...item, selected: true })
    }
    
    saveToStorage()
  }
  
  function removeItem(skuId: string) {
    const index = items.value.findIndex(i => i.skuId === skuId)
    if (index > -1) {
      items.value.splice(index, 1)
    }
    saveToStorage()
  }
  
  function updateQuantity(skuId: string, quantity: number) {
    const item = items.value.find(i => i.skuId === skuId)
    if (item) {
      item.quantity = Math.max(1, quantity)
    }
    saveToStorage()
  }
  
  function toggleSelect(skuId: string) {
    const item = items.value.find(i => i.skuId === skuId)
    if (item) {
      item.selected = !item.selected
    }
    saveToStorage()
  }
  
  function selectAll(selected: boolean) {
    items.value.forEach(item => item.selected = selected)
    saveToStorage()
  }
  
  function clear() {
    items.value = []
    saveToStorage()
  }
  
  function saveToStorage() {
    storage.set('cart_items', items.value)
  }
  
  function loadFromStorage() {
    items.value = storage.get<CartItem[]>('cart_items') || []
  }
  
  return {
    items,
    totalPrice,
    totalCount,
    selectedCount,
    addItem,
    removeItem,
    updateQuantity,
    toggleSelect,
    selectAll,
    clear,
    loadFromStorage
  }
})
```
