# MOYUYO uni-app + 原生插件混合架构方案

## 一、架构概述

### 1.1 设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                        架构分层原则                              │
├─────────────────────────────────────────────────────────────────┤
│  ① 单一职责：每个模块只负责一个业务领域                          │
│  ② 接口隔离：uni-app 与原生通过清晰的接口通信                    │
│  ③ 渐进增强：3D/AR 作为原生插件，不影响主链路                    │
│  ④ 降级兼容：小程序/H5 端提供 3D 降级方案                       │
│  ⑤ 数据同步：共享后端 API，前端状态统一管理                      │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 模块划分

| 模块 | 技术方案 | 说明 |
|:---|:---|:---|
| **电商主链路** | uni-app（Vue 3） | 商品、购物车、订单、支付、会员、社区、消息 |
| **3D Pet Hub** | Android 原生插件（Filament）+ iOS 原生插件（SceneKit） | 3D 渲染、手势交互、IP 动画 |
| **AR 试穿** | Android 原生插件（ARCore）+ iOS 原生插件（ARKit） | AR 追踪、试穿效果 |
| **支付 SDK** | 原生插件封装 | Stripe、PayPal、Apple Pay、Google Pay |
| **社交登录** | 原生插件封装 | Apple Sign-In、Google、Facebook |
| **推送通知** | 原生插件封装 | FCM、APNs |

## 二、目录结构

### 2.1 项目根目录

```
MOYUYO/
├── uni-app/                    # uni-app 主项目
│   ├── src/                    # 源代码
│   │   ├── pages/              # 页面
│   │   │   ├── home/           # 首页
│   │   │   ├── product/        # 商品列表/详情
│   │   │   ├── cart/           # 购物车
│   │   │   ├── order/          # 订单管理
│   │   │   ├── community/      # 社区
│   │   │   ├── member/         # 会员中心
│   │   │   ├── message/        # 消息中心
│   │   │   ├── support/        # 客服
│   │   │   └── pet-hub/        # Pet Hub 入口页面（调用原生插件）
│   │   ├── components/         # 公共组件
│   │   ├── stores/             # Pinia 状态管理
│   │   ├── utils/              # 工具函数
│   │   ├── api/                # API 封装
│   │   ├── plugins/            # 原生插件调用封装
│   │   └── static/             # 静态资源
```
