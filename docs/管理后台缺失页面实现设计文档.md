# 管理后台缺失页面实现设计文档

> 文档版本：V1.0.0
> 编写日期：2026-07-17
> 适用范围：MOYUYO 管理后台（moyuyo-admin）缺失页面补充实现

---

## 一、概述

基于 `APPdocs/admin/` 设计稿与 `moyuyo-server/moyuyo-admin/` 现有实现的一致性检查，**68 个设计稿页面中有 54 个尚未实现**。本文档规划如何分批补充这些缺失页面。

### 1.1 技术栈

| 项目 | 选型 | 说明 |
|:---|:---|:---|
| 框架 | Vue 3 + Vite | 与现有项目一致 |
| UI 组件库 | **Element Plus** | 新增引入，提升开发效率 |
| 路由 | Vue Router 4 | 现有项目已使用 |
| HTTP 客户端 | Axios | 现有项目已使用（先占位，后续对接） |
| 样式 | 现有设计令牌 + Element Plus 主题定制 | 保持视觉一致性 |

### 1.2 原则

- **不重构现有 14 个页面**：避免破坏已有功能
- **新页面统一使用 Element Plus**：表格、表单、分页、对话框等从 Element Plus 导入
- **先模拟数据，后对接接口**：每个页面先用组件内模拟数据占位，后续替换为 API 调用
- **保持设计语言一致**：通过 CSS 变量覆盖 Element Plus 主题色，与现有 Pinguo 设计令牌对齐

## 二、实施阶段

### 第一阶段：基础设施搭建 + 核心运营模块（6 页）

**步骤 1：安装 Element Plus**

```bash
cd moyuyo-server/moyuyo-admin
npm install element-plus @element-plus/icons-vue
```

**步骤 2：全局注册 Element Plus**

修改 `src/main.js`，注册 Element Plus 和图标。

**步骤 3：主题定制**

在 `src/styles/` 下新建 `element-overrides.css`，将现有设计令牌映射到 Element Plus 的 CSS 变量：

- `--el-color-primary` → `--brand-500 (#007aff)`
- `--el-color-success` → `--state-success (#34c759)`
- `--el-color-danger` → `--state-error (#ff3b30)`
- `--el-border-radius-base` → `--radius (12px)`

## 三、页面模板规范

每个新页面遵循以下模板：

```vue
<template>
  <div class="page-wrapper">
    <!-- 页面标题 -->
    <div class="page-header">
      <h2>{{ pageTitle }}</h2>
      <div class="header-actions">
        <el-button type="primary">新建</el-button>
      </div>
    </div>

    <!-- 搜索/筛选区域 -->
    <el-card shadow="never" class="filter-card">
      <el-form :model="filters" inline>
        <el-form-item label="搜索">
          <el-input v-model="filters.keyword" placeholder="请输入关键词" />
        </el-form-item>
        <el-form-item>
          <el-button type="primary" @click="handleSearch">搜索</el-button>
          <el-button @click="handleReset">重置</el-button>
        </el-form-item>
      </el-form>
    </el-card>

    <!-- 表格区域 -->
    <el-card shadow="never">
      <el-table :data="tableData" stripe>
        <el-table-column prop="id" label="ID" width="80" />
        <el-table-column prop="name" label="名称" />
        <el-table-column label="状态">
          <template #default="{ row }">
            <el-tag :type="row.status === 'active' ? 'success' : 'info'">
              {{ row.status === 'active' ? '启用' : '禁用' }}
            </el-tag>
          </template>
        </el-table-column>
        <el-table-column label="操作" width="200">
          <template #default="{ row }">
            <el-button size="small" @click="handleEdit(row)">编辑</el-button>
            <el-button size="small" type="danger" @click="handleDelete(row)">删除</el-button>
          </template>
        </el-table-column>
      </el-table>
      <el-pagination
        v-model:current-page="currentPage"
        v-model:page-size="pageSize"
        :total="total"
        layout="total, sizes, prev, pager, next"
        @change="loadData"
      />
    </el-card>

    <!-- 编辑对话框 -->
    <el-dialog v-model="dialogVisible" :title="dialogTitle" width="600px">
      <el-form :model="editForm" label-width="100px">
        <el-form-item label="名称">
          <el-input v-model="editForm.name" />
        </el-form-item>
      </el-form>
      <template #footer>
        <el-button @click="dialogVisible = false">取消</el-button>
        <el-button type="primary" @click="handleSave">保存</el-button>
      </template>
    </el-dialog>
  </div>
</template>

<script setup>
import { ref, reactive } from 'vue'
import { ElMessage, ElMessageBox } from 'element-plus'

const pageTitle = '页面标题'

// 筛选条件
const filters = reactive({ keyword: '' })

// 表格数据（模拟数据，后续替换为 API）
const tableData = ref([])
const currentPage = ref(1)
const pageSize = ref(15)
const total = ref(0)

// 对话框
const dialogVisible = ref(false)
const dialogTitle = ref('')
const editForm = reactive({})

function loadData() {
  // 模拟数据加载
  tableData.value = []
  total.value = 0
}

function handleSearch() {
  currentPage.value = 1
  loadData()
}

function handleReset() {
  filters.keyword = ''
  handleSearch()
}

function handleEdit(row) {
  dialogTitle.value = '编辑'
  Object.assign(editForm, row)
  dialogVisible.value = true
}

function handleDelete(row) {
  ElMessageBox.confirm('确定删除？', '提示').then(() => {
    ElMessage.success('删除成功')
    loadData()
  })
}

function handleSave() {
  ElMessage.success('保存成功')
  dialogVisible.value = false
  loadData()
}

loadData()
</script>

<style scoped>
.page-wrapper {
  padding: 20px;
}

.page-header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  margin-bottom: 20px;
}

.page-header h2 {
  font-size: 20px;
  font-weight: 700;
  color: var(--text-800);
}

.filter-card {
  margin-bottom: 16px;
}
</style>
```

## 四、路由注册方案

所有新页面路由在 `router/index.js` 中统一注册，作为 AdminLayout 的子路由：

```javascript
{
  path: 'cms',
  name: 'CmsManage',
  component: () => import('../views/CmsManage.vue')
}
```

侧边栏导航在 `AdminLayout.vue` 的 `navItems` 数组中追加新条目：

```javascript
{ path: '/cms', icon: '📄', label: 'CMS管理' },
{ path: '/rbac', icon: '🔐', label: '权限管理' },
// ... 其他页面
```

由于页面量大，为了不使 `navItems` 过长，侧边栏将采用**分组折叠**的方式：

| 分组 | 图标 | 子页面 |
|:---|:---|:---|
| 商品运营 | 📦 | 商品分析、商品报表、价格管理、价格历史 |
| 订单运营 | 🛒 | 订单导出、订单拦截、订单监控、订单改价、订单打印 |
| 营销运营 | 📢 | 活动创建、营销效果、A/B测试 |
| 物流仓储 | 🚚 | 仓库管理、海外仓、合包/分包裹、承运商对比 |
| 系统管理 | ⚙️ | CMS、RBAC、系统配置、敏感词、审计日志、操作日志 |
| 安全合规 | 🔒 | GDPR、风控管理、风控规则引擎 |
| 数据洞察 | 📊 | RFM分析、漏斗分析、用户画像、搜索分析、流量分析 |
| 运营工具 | 🛠️ | 推送管理、短信管理、知识库、批量导入、直播管理、满意度管理 |
