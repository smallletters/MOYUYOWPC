# MOYUYO 1Panel 部署指南

> 版本：V1.0.0  
> 日期：2026-07-07  
> 适用场景：MVP 阶段单服务器部署  
> 服务器要求：Ubuntu 22.04 LTS / 2核4GB / 40GB SSD

---

## 一、目录结构

```
deploy/
├── init-server.sh          # 服务器初始化脚本
├── docker-compose.yml      # Docker Compose 配置
├── .env.example            # 环境变量模板
├── .env                    # 环境变量（需手动创建）
├── redis.conf              # Redis 配置
├── nginx/
│   ├── nginx.conf          # Nginx 主配置
│   └── conf.d/
│       └── moyuyo.conf     # MOYUYO 站点配置
├── init-sql/
│   └── init-db.sql         # 数据库初始化脚本
└── ssl/                    # SSL 证书目录（1Panel 自动生成）
```

## 二、部署步骤

### 步骤 1：准备服务器

**推荐配置**：
- CPU：2 核
- 内存：4GB
- 存储：40GB SSD
- 带宽：3Mbps+
- 系统：Ubuntu 22.04 LTS

**购买渠道**：阿里云轻量应用服务器、腾讯云轻量服务器、华为云弹性云服务器等。

### 步骤 2：配置域名解析

| 域名 | 类型 | 值 | 说明 |
| :--- | :--- | :--- | :--- |
| api.moyuyo.com | A | <服务器IP> | API 接口域名 |
| www.moyuyo.com | A | <服务器IP> | 前端站点域名 |
| moyuyo.com | A | <服务器IP> | 主域名 |

> 建议使用 Cloudflare DNS，可获得免费 DDoS 防护和 CDN 加速。

### 步骤 3：初始化服务器

```bash
# 1. 登录服务器
ssh root@your-server-ip

# 2. 下载部署文件（假设已上传到服务器）
cd /opt
git clone https://github.com/moyuyo/deploy.git moyuyo-deploy
cd moyuyo-deploy

# 3. 执行初始化脚本
bash init-server.sh
```

**脚本执行内容**：
- 更新系统和基础依赖
- 安装 Docker 和 Docker Compose
- 安装 1Panel 面板
- 配置防火墙规则
- 创建目录结构
- 设置时区

### 步骤 4：配置 1Panel

1. **访问 1Panel**：`http://<服务器IP>:443`
2. **设置管理员账号**：创建用户名和密码
3. **配置系统设置**：
   - 系统语言：中文
   - 时区：Asia/Shanghai
   - 邮件通知（可选）

### 步骤 5：创建环境变量文件

```bash
cd /opt/moyuyo-deploy
cp .env.example .env

# 编辑环境变量文件
vim .env
```

**必须修改的配置项**：

| 配置项 | 说明 | 示例 |
| :--- | :--- | :--- |
| MYSQL_ROOT_PASSWORD | MySQL 根密码 | 复杂密码 |
| MYSQL_PASSWORD | MOYUYO 数据库密码 | 复杂密码 |
| REDIS_PASSWORD | Redis 密码 | 复杂密码 |
| JWT_SECRET | JWT 密钥 | 至少 32 位随机字符串 |
| STRIPE_API_KEY | Stripe 密钥 | sk_xxx |
| PAYPAL_CLIENT_ID | PayPal Client ID | xxx |
| AWS_ACCESS_KEY | AWS 密钥 | xxx |
| SENDGRID_API_KEY | SendGrid 密钥 | SG.xxx |

### 步骤 6：启动服务

```bash
cd /opt/moyuyo-deploy

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f moyuyo-backend
```

**服务启动顺序**：
1. MySQL → 健康检查通过
2. Redis → 健康检查通过
3. RocketMQ NameServer → 健康检查通过
4. RocketMQ Broker → 依赖 NameServer
5. MOYUYO Backend → 依赖 MySQL/Redis/RocketMQ
6. Nginx → 依赖 Backend

### 步骤 7：配置 SSL 证书

**方法一：1Panel 自动签发（推荐）**

1. 登录 1Panel → 网站 → 添加站点
2. 域名：api.moyuyo.com
3. 类型：反向代理
4. 反向代理地址：http://localhost:8080
5. SSL 证书：勾选「自动签发 Let's Encrypt 证书」

**方法二：手动配置**

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx

# 签发证书
certbot certonly --nginx -d api.moyuyo.com -d www.moyuyo.com

# 证书续期
certbot renew --dry-run
```

### 步骤 8：验证部署

```bash
# 1. 健康检查
curl http://localhost:8080/api/health

# 预期响应：
# {"status":"UP","timestamp":"2026-07-07T10:00:00","version":"1.0.0"}

# 2. API 测试
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@moyuyo.com","password":"password"}'

# 3. Nginx 测试
curl https://api.moyuyo.com/api/health
```

## 三、1Panel 管理

### 3.1 常用操作

| 操作 | 路径 | 说明 |
| :--- | :--- | :--- |
| 查看容器 | 容器管理 | 查看/启动/停止/删除容器 |
| 查看日志 | 容器管理 → 日志 | 查看各服务日志 |

1. 登录 1Panel → 数据库管理
2. 选择 moyuyo_prod 数据库
3. 点击「备份」→ 选择备份方式（全量/增量）
4. 设置备份周期（建议每日凌晨 2 点）

### 3.3 日志查看

```bash

tail -f /opt/moyuyo-deploy/nginx/logs/access.log

# Nginx 错误日志
tail -f /opt/moyuyo-deploy/nginx/logs/error.log
```

## 四、升级部署

### 4.1 升级后端应用

```bash
cd /opt/moyuyo-deploy

# 1. 更新环境变量中的版本号
vim .env
# 修改 DOCKER_TAG=v1.1.0

# 2. 拉取新镜像
docker-compose pull moyuyo-backend

# 3. 重启服务
docker-compose up -d moyuyo-backend

# 4. 验证
curl https://api.moyuyo.com/api/health
```

### 4.2 升级数据库

```bash
# 1. 创建新的迁移脚本
vim deploy/init-sql/V8__new_feature.sql

# 2. 执行迁移
docker-compose exec mysql mysql -u moyuyo -p moyuyo_prod < V8__new_feature.sql
```

## 五、监控与告警

### 5.1 基础监控

**1Panel 仪表盘**：
- CPU 使用率
- 内存使用率
- 磁盘使用率
- 网络流量
- 容器状态

### 5.2 自建监控（可选）

```yaml
# docker-compose.yml 添加 Prometheus 和 Grafana
prometheus:
  image: prom/prometheus:latest
  container_name: moyuyo-prometheus
  ports:
    - "9090:9090"
  volumes:
    - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
    - /opt/moyuyo/data/prometheus:/prometheus
  restart: unless-stopped

grafana:
  image: grafana/grafana:latest
  container_name: moyuyo-grafana
  ports:
    - "3000:3000"
  volumes:
    - /opt/moyuyo/data/grafana:/var/lib/grafana
  environment:
    - GF_SECURITY_ADMIN_PASSWORD=admin
  restart: unless-stopped
```

### 5.3 告警规则

| 告警项 | 阈值 | 通知方式 |
| :--- | :--- | :--- |
| CPU > 80% | 持续 5 分钟 | 邮件/企业微信 |
| 内存 > 85% | 持续 5 分钟 | 邮件/企业微信 |
| 磁盘 > 90% | 持续 5 分钟 | 邮件/企业微信 |
| 应用不可用 | 连续 3 次健康检查失败 | 邮件/企业微信 |
| 数据库连接数 > 80% | 持续 5 分钟 | 邮件/企业微信 |

## 六、安全最佳实践

### 6.1 服务器安全

| 措施 | 说明 |
| :--- | :--- |
| 禁用 root 远程登录 | 修改 `/etc/ssh/sshd_config` |
| 使用密钥登录 | 创建 SSH 密钥对 |
| 更改 SSH 端口 | 修改 `/etc/ssh/sshd_config` |
| 配置防火墙 | 使用 ufw 限制访问 |
| 定期更新系统 | 每周执行 `apt update && apt upgrade` |

### 6.2 应用安全

| 措施 | 说明 |
| :--- | :--- |
| 启用 HTTPS | 使用 Let's Encrypt 证书 |
| 配置 WAF | 使用 Cloudflare WAF |
| 限制 API 访问 | Nginx 限流配置 |
| 隐藏服务器信息 | 关闭 server_tokens |
| 防 SQL 注入 | 后端参数校验 |

### 6.3 数据安全

| 措施 | 说明 |
| :--- | :--- |
| 数据库密码 | 复杂密码，定期更换 |
| 数据备份 | 每日自动备份，异地存储 |
| 敏感数据加密 | 密码 BCrypt 哈希，Token AES 加密 |
| 日志脱敏 | 敏感字段自动脱敏 |

## 七、故障排查

### 7.1 常见问题

| 问题 | 原因 | 解决方法 |
| :--- | :--- | :--- |
| 服务启动失败 | 端口被占用 | `netstat -tlnp | grep <端口>` 查看并释放 |
| 数据库连接失败 | 密码错误或网络不通 | 检查 `.env` 配置，测试连通性 |
| API 响应超时 | 后端服务未启动 | `docker-compose logs moyuyo-backend` |
| SSL 证书过期 | 未自动续期 | `certbot renew` 手动续期 |
| 内存不足 | 应用占用过高 | 增加服务器内存或优化应用 |

### 7.2 日志排查

```bash
# 查看所有服务日志
docker-compose logs --tail=100

# 查看特定服务日志
docker-compose logs -f moyuyo-backend

# 查看 MySQL 慢查询日志
docker-compose exec mysql mysql -e "SHOW LOGS;"

# 查看 Redis 慢查询日志
docker-compose exec redis redis-cli slowlog get 10
```

## 八、性能优化建议

### 8.1 数据库优化

| 优化项 | 说明 |
| :--- | :--- |
| 索引优化 | 为高频查询字段创建索引 |
| 查询优化 | 避免 SELECT *，使用 LIMIT |
| 连接池 | 配置合理的连接池大小 |
| 读写分离 | MVP 阶段暂不需要 |

### 8.2 缓存优化

| 优化项 | 说明 |
| :--- | :--- |
| 热点数据缓存 | 商品详情、用户信息缓存 |
| 缓存策略 | 合理设置 TTL，避免缓存雪崩 |
| 缓存预热 | 启动时预热热门数据 |

### 8.3 Nginx 优化

| 优化项 | 说明 |
| :--- | :--- |
| Gzip 压缩 | 启用 gzip 减少传输体积 |
| 静态资源缓存 | 设置合理的缓存时间 |
| 连接复用 | 调整 keepalive 参数 |

## 九、扩容建议

### 9.1 垂直扩容

当单服务器资源不足时：
1. 升级服务器配置（2核4GB → 4核8GB → 8核16GB）
2. 优化应用性能（代码优化、SQL 优化）

### 9.2 水平扩容（增长期）

当单服务器无法满足需求时：
1. 数据库迁移到云数据库（阿里云 RDS）
2. Redis 迁移到云缓存（阿里云 Redis）
3. 应用部署到多台服务器，配置负载均衡
4. 使用 Kubernetes 进行容器编排
