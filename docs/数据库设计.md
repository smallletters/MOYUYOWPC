## 二、表结构详细设计

### 2.1 用户表 `mo_user`

| 字段 | 类型 | 索引 | 说明 |
| :--- | :--- | :---: | :--- |
| id | bigint | PK | 雪花 ID |
| **email** | varchar(128) | UK | **邮箱**（欧美主键） |
| phone | varchar(20) | IDX | 手机号（备选，部分国家使用） |
| **password_hash** | varchar(128) | | **密码 BCrypt 加盐**（仅邮箱注册用户） |
| nickname | varchar(64) | | 昵称 |
| avatar | varchar(255) | | 头像 URL |
| gender | tinyint | | 0 未知 1 男 2 女 |
| birthday | date | | 出生日期（用于年龄校验） |
| **country** | varchar(8) | IDX | **国家代码 ISO 3166-1 alpha-2（US/GB/DE/FR...）** |
| **locale** | varchar(16) | | **区域（en_US / en_GB / de_DE...）** |
| **timezone** | varchar(64) | | **时区（IANA：America/Los_Angeles...）** |
| member_level | varchar(16) | IDX | NORMAL/SILVER/GOLD/DIAMOND/BLACK |
| growth_value | int | | 成长值 |
| points | int | | 积分 |
| **two_factor_enabled** | tinyint | | **2FA 是否开启** |
| **email_verified** | tinyint | | **邮箱是否验证** |
| **marketing_opt_in** | tinyint | | **营销邮件订阅（GDPR 要求显式同意）** |
| status | tinyint | | 0 正常 1 封禁 2 注销中 3 已注销 |
| register_time | datetime | | 注册时间 |
| last_login_time | datetime | | 最近登录 |
| **delete_scheduled_at** | datetime | | **账号注销计划执行时间** |

**索引**
- `uk_email` UNIQUE(`email`)
- `idx_phone`(`phone`)
- `idx_country`(`country`)
- `idx_status_delete`(`status`,`delete_scheduled_at`)

**字段变更说明（从旧版迁移）**
- 删除 `phone UK` 唯一约束，改为普通索引。
- 保留 `phone` 字段兼容老用户，但新注册用户主键为 `email`。
- 密码字段从可选改为必填（邮箱注册用户）。

### 2.1.1 第三方登录绑定表 `mo_oauth_binding`

| 字段 | 类型 | 索引 | 说明 |
| :--- | :--- | :---: | :--- |
| id | bigint | PK | |
| user_id | bigint | IDX | 关联 mo_user |
| **provider** | varchar(16) | IDX | **APPLE / GOOGLE / FACEBOOK** |
| **provider_uid** | varchar(128) | UK(provider, provider_uid) | 第三方平台唯一标识 |
| **provider_email** | varchar(128) | | 第三方返回的邮箱（可能为空） |
| **raw_profile** | json | | 第三方返回的原始用户信息（脱敏） |
| bind_time | datetime | | 绑定时间 |
| last_login_time | datetime | | 最近通过此方式登录 |

**索引**
- `uk_provider_uid` UNIQUE(`provider`,`provider_uid`)
- `idx_user_provider`(`user_id`,`provider`)

**业务规则**
- 一个用户可绑定多个第三方账号（如同时绑 Google + Facebook）。
- 同一 provider_uid 只能绑定一个 user。
- 首次第三方登录若邮箱已存在，自动合并到现有账号（需二次确认）。

### 2.1.3 同意日志表 `mo_consent_log`（GDPR 必需）

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | bigint PK | |
| user_id | bigint | 用户 |
| **consent_type** | varchar(32) | **PRIVACY_POLICY / TERMS / MARKETING_EMAIL / ANALYTICS_COOKIE / ADVERTISING_COOKIE** |
| granted | tinyint | 0 拒绝 1 同意 |
| **version** | varchar(16) | **协议版本号（条款更新时递增）** |

| revoke_time | datetime | 撤销时间（同意 → 拒绝） |

**业务规则**
- 隐私政策更新时必须记录新版本，所有用户需重新同意。
- 审计保留 5 年（GDPR 要求可证明合规）。

### 2.1.5 账号注销表 `mo_account_deletion`

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | bigint PK | |
| user_id | bigint UK | 一对一 |
| status | varchar(16) | PENDING / RECOVERED / EXECUTED |
| reason | varchar(32) | 注销原因枚举 |
| scheduled_at | datetime | 计划执行时间（申请时间 + 30 天） |
| recover_deadline | datetime | 可恢复截止时间 |

- 宽限期后由定时任务将 status 改为 EXECUTED，并执行物理删除 / 匿名化。

### 2.1.7 设备管理表 `mo_device`

| 字段 | 类型 | 索引 | 说明 |
| :--- | :--- | :---: | :--- |
| id | bigint | PK | 雪花 ID |
| user_id | bigint | IDX | 用户 ID |
| device_id | varchar(64) | | 设备唯一标识 |
| platform | varchar(16) | | iOS / Android / Web |
| model | varchar(64) | | 设备型号 |
| os_version | varchar(32) | | 操作系统版本 |
| app_version | varchar(32) | | App 版本 |
| push_token | varchar(255) | | 推送 Token |
| trusted | tinyint | | 0 普通设备 1 2FA 可信设备 |
| last_active | datetime | | 最后活跃时间 |
| create_time | datetime | | 首次绑定时间 |

**业务规则**
- 同一用户最多绑定 3 台设备同时在线，超出时自动踢出最早设备。
- 可信设备在登录时跳过 2FA 验证。

## 四、索引规范

1. **主键统一**：`bigint` + 雪花算法（`Snowflake`）。
2. **唯一索引**：业务唯一字段（手机号、订单号、SKU 编码）。
3. **复合索引**：遵循最左前缀原则，按"高频查询字段"前置。
4. **避免过度索引**：单表索引不超过 6 个。
5. **大字段不进索引**：JSON、Text 字段不建索引。
6. **数据量过大表**：按时间分区或归档（如订单 1 年前数据归档至冷库）。

### 6.1 注销宽限期机制

```
用户申请注销
   ↓
创建 mo_account_deletion（status=PENDING，scheduled_at=now+30d）
   ↓
```

### 6.4 定时任务（每日 03:30 执行）

```java
// AccountDeletionJob.java（伪代码）
@Scheduled(cron = "0 30 3 * * ?")
public void executeScheduledDeletions() {
  List<AccountDeletion> pendings = accountDeletionRepository
      .findByStatusAndScheduledAtBefore("PENDING", LocalDateTime.now());
  for (AccountDeletion deletion : pendings) {
    long userId = deletion.getUserId();
    // 1. 物理删除 PII 表
    petRepository.deleteByUserId(userId);
    addressRepository.deleteByUserId(userId);
    cartRepository.deleteByUserId(userId);
    oauthBindingRepository.deleteByUserId(userId);
    twoFactorRepository.deleteByUserId(userId);
    // 2. 订单匿名化
    orderService.anonymizeOrders(userId);
    // 3. 删除用户主表
    userRepository.deleteById(userId);
    // 4. 更新注销状态
    deletion.setStatus("EXECUTED");
    deletion.setExecutedAt(LocalDateTime.now());
    accountDeletionRepository.save(deletion);
  }
}
```
